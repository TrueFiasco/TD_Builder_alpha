# Meta-Skill: TouchDesigner Skill Builder

## Purpose

Create reusable skills (parameterized tool chains) for common TD operations. Skills are smaller than agents - they're specific tool sequences that can be invoked as single actions.

## How to Use

Provide the following inputs to generate a skill:

```yaml
skill_name: "validate_td_network"
skill_description: "Validate TouchDesigner network JSON against all rules"
parameters:
  - name: "network_json"
    type: "object"
    required: true
    description: "Network specification to validate"
  - name: "verbose"
    type: "boolean"
    default: false
    description: "Include detailed validation messages"
tool_chain:
  - step: 1
    tool: "td_validate"
    inputs: ["network_json", "verbose"]
  - step: 2
    condition: "step1.result.valid == false"
    tool: "td_assistant"
    inputs: ["Fix suggestion query"]
```

## Generated Skill Structure

### skill_{name}.yaml

```yaml
# Skill: {skill_name}
# Auto-generated by meta_skill.md

name: "{skill_name}"
description: "{skill_description}"
version: "1.0"
created: "{timestamp}"

# Input parameters
parameters:
  {for param in parameters}
  - name: "{param.name}"
    type: "{param.type}"
    required: {param.required}
    default: {param.default}
    description: "{param.description}"
  {endfor}

# Execution steps
execution:
  {for step in tool_chain}
  - step: {step.step}
    tool: "{step.tool}"
    inputs:
      {for input in step.inputs}
      - "{input}"
      {endfor}
    {if step.condition}
    condition: "{step.condition}"
    {endif}
    timeout_seconds: 60
    retry_on_failure: false
  {endfor}

# Expertise integration
expertise_integration:
  pre_check:
    - file: "td_problems.yaml"
      check: "No unresolved critical problems for this skill"
  post_update:
    - file: "td_network_building.yaml"
      update_type: "increment_counter"
      on_success: "working_patterns.{pattern}.success_count"
      on_failure: "failing_patterns.{pattern}.failure_count"

# Output specification
output:
  format: "yaml"
  schema:
    success: boolean
    result: object
    errors: array
    metrics:
      duration_ms: number
      steps_completed: number
```

## Common TD Skills

### 1. Network Validation Skill

```yaml
skill_name: "validate_network"
skill_description: "Validate TD network against schema and rules"
parameters:
  - name: "network"
    type: "object"
    required: true
  - name: "strict"
    type: "boolean"
    default: true
tool_chain:
  - step: 1
    tool: "schema_validate"
    inputs: ["network"]
  - step: 2
    tool: "operator_validate"
    inputs: ["network.operators"]
  - step: 3
    tool: "connection_validate"
    inputs: ["network.connections"]
```

### 2. Pattern Match Skill

```yaml
skill_name: "match_pattern"
skill_description: "Match network to known workflow patterns"
parameters:
  - name: "network"
    type: "object"
    required: true
tool_chain:
  - step: 1
    tool: "load_expertise"
    inputs: ["td_network_patterns.yaml"]
  - step: 2
    tool: "pattern_compare"
    inputs: ["network", "expertise.workflows"]
  - step: 3
    tool: "confidence_score"
    inputs: ["match_results"]
```

### 3. Summary Generation Skill

```yaml
skill_name: "generate_summary"
skill_description: "Generate LLM-friendly summary from semantic JSON"
parameters:
  - name: "semantic_json"
    type: "object"
    required: true
  - name: "curator_text"
    type: "string"
    required: false
tool_chain:
  - step: 1
    tool: "load_expertise"
    inputs: ["td_operators.yaml", "td_network_patterns.yaml"]
  - step: 2
    tool: "identify_pattern"
    inputs: ["semantic_json"]
  - step: 3
    tool: "extract_dataflow"
    inputs: ["semantic_json"]
  - step: 4
    tool: "generate_summary"
    inputs: ["pattern", "dataflow", "curator_text"]
  - step: 5
    tool: "validate_summary"
    inputs: ["summary", "semantic_json"]
```

### 4. Build Network Skill

```yaml
skill_name: "build_network"
skill_description: "Build TD network from specification"
parameters:
  - name: "spec"
    type: "object"
    required: true
  - name: "output_format"
    type: "string"
    default: "json"
tool_chain:
  - step: 1
    tool: "load_expertise"
    inputs: ["td_network_building.yaml"]
  - step: 2
    tool: "validate_spec"
    inputs: ["spec"]
  - step: 3
    condition: "step2.valid"
    tool: "build"
    inputs: ["spec"]
  - step: 4
    tool: "convert_format"
    inputs: ["built_network", "output_format"]
```

### 5. Problem Report Skill

```yaml
skill_name: "report_problem"
skill_description: "Log problem with root cause analysis"
parameters:
  - name: "error"
    type: "object"
    required: true
  - name: "context"
    type: "object"
    required: true
tool_chain:
  - step: 1
    tool: "analyze_root_cause"
    inputs: ["error", "context"]
  - step: 2
    tool: "match_problem_pattern"
    inputs: ["error", "td_problems.yaml"]
  - step: 3
    tool: "generate_problem_id"
    inputs: []
  - step: 4
    tool: "log_problem"
    inputs: ["problem_id", "error", "root_cause", "context"]
```

## Skill Composition

Skills can be composed into higher-level workflows:

```yaml
workflow: "analyze_and_summarize"
steps:
  - skill: "validate_network"
    inputs: { network: "$input.network" }
  - skill: "match_pattern"
    inputs: { network: "$input.network" }
    condition: "$step1.success"
  - skill: "generate_summary"
    inputs:
      semantic_json: "$input.network"
      curator_text: "$input.curator"
    condition: "$step2.confidence > 0.5"
```

## Skill Metrics

Each skill execution tracks:

```yaml
metrics:
  execution_id: "unique-id"
  skill_name: "validate_network"
  start_time: "ISO timestamp"
  end_time: "ISO timestamp"
  duration_ms: N
  success: true|false
  steps_completed: N
  steps_total: N
  errors: []
```

## Integration with Expertise

Skills can update expertise after execution:

```yaml
expertise_updates:
  on_success:
    - file: "td_network_building.yaml"
      action: "increment"
      path: "build_history"
      value:
        timestamp: "{now}"
        skill: "{skill_name}"
        result: "success"

  on_failure:
    - file: "td_problems.yaml"
      action: "log_if_new"
      content:
        category: "skill_failure"
        skill: "{skill_name}"
        error: "{error_message}"
```
