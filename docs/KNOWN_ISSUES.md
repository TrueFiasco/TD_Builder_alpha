# Known Issues — V0.2.0

Documented limitations, not blockers. The 22-check acceptance + smoke gate passes; KB search is
local / key-free.

| Area | Issue | Status / workaround |
|---|---|---|
| **Palette components** | Per-operator `palette` fields build an **external-tox placeholder** that loads the component from the **user's own TD install** at open time (`app.samplesFolder` expression + `subcompname`) — no Derivative content is bundled. 277 items registered in `KB/palette_components.json`; wires to/from the placeholder survive loading (TD-native `compinputs` + inner-out-op serialization). | **Works.** Caveats: components mirror the palette of the TD version they load on (registry harvested on 2025.32820); an item missing from the user's install loads empty (no crash). Design-level `{"palette": …}` keys are rejected with a pointer to the per-operator form. For unregistered `.tox` files use `external_tox`. |
| **Component as a data source** | A component is never itself a data source — what it *outputs* lives on its inner `out` op (`comp/out1`), separate from what it *displays*. Two effects this release: (a) a build that wires a user-supplied **`external_tox`** component as a connection source/target silently **drops** that wire (registered `palette` items and in-design containers wire correctly); (b) an expression like `op('comp')['sin']` errors at runtime (`baseCOMP not subscriptable`). | **Workaround:** reference the inner out op — `comp/out1` in a connection, `op('comp/out1')['sin']` in an expression — or hand-wire in TD. Builder `external_tox` resolution + a validation rule + the guidance fix are scheduled for the next release. |
| **BASIC-mode build** | Building a network from scratch (no parsed source) emits parameter-format warnings; some expression modes are approximate. | The **LOSSLESS** round-trip (parse an existing `.toe.dir` → rebuild) is byte-accurate — use it when you have a source. |
| `find_similar_networks` | Returns `[]` for many example ids (partial pattern coverage). | All other KB tools are full-fidelity. Deferred. |
| `hybrid_search.relationships` | Some display-name vs graph-key mismatches (e.g. "Feedback TOP") report "not found in graph". | Semantic results are unaffected; graph node-name normalization deferred. |
| `get_network_patterns` | Thin/noisy at high `min_frequency`. | Use a lower `min_frequency`; co-occurrence dedup deferred. |
| Final `.toe` file | `td_build_project(mode="toe")` builds a real collapsed single `.toe` (runs `toecollapse` internally) alongside its `.toe.dir` + `.toc`; `mode="tox"` builds a `.tox` component. | **Works** (needs TD's `toecollapse`, which ships with TD). |
| Live tools | Require TouchDesigner running + `MCP/td-webserver/mcp_webserver_base.tox` imported (port 9981). | Without TD, the 19 `td-builder-live` tools return a clear "not running" message. |
| **Live-server auth** | The WebServer DAT binds all interfaces (cannot be loopback-only). Requests need `Authorization: Bearer <token>`; missing/invalid → **401** with a self-describing hint. | TD generates the token on first boot at `~/.td_builder/api_token` and prints only the path. Same machine = zero config (the client reads it). Remote client = copy the value, set `TD_API_TOKEN` in the client env. Rotate = delete the file + **restart TD**. `GET /api/td/server/td` stays open (health). See `MCP/COMM_LAYER.md`. |
| **Live-server transport** | Token + payloads travel **plaintext HTTP** (no TLS this release). | Token auth is the LAN control; for untrusted networks tunnel via SSH/VPN or stay on `127.0.0.1`. Debug logging is OFF by default (`TD_API_DEBUG=1`); capture payloads are never logged. |
| **Multi-expert orchestration** | The `get_expert_prompt` prompts describe a plan → build → critic pipeline, but this release has no automated orchestrator — you fetch and run each expert prompt individually. | **Deferred to a later release.** Prompts now route all knowledge through the MCP tools; the cross-expert pipeline is manual for now. |
| **Self-improvement / curated expertise** | The experts' `self_improve` phase and the 18 curated `Agents/expertise/*.yaml` are not wired into the runtime (only `td_network_building.yaml` is read by code). | **Deferred to a later release.** Those YAMLs plus three currently-unused engine modules (`engine/writers/lossless_writer.py`, `server_core/search/__init__.py`, `Agents/experts/__init__.py`) are kept for a future "connect in or remove" decision. |

**No API key anywhere** — KB semantic search uses a local embedding model. The first KB-dependent
call warms the knowledge base (~1–2 min, one-time), then every call is fast.
