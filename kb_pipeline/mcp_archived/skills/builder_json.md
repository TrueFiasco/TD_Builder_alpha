# Skill: builder_json

## Purpose
Create or modify an expanded TouchDesigner project (`.toe.dir`/`.tox.dir` + `.toc`) so it can be collapsed with `toecollapse.exe` into a working `.toe`/`.tox`.

This skill produces file-system actions and a deterministic build plan. It does **not** rely on manual TouchDesigner UI steps.

## Inputs
- `TDNetworkSpec` (required): nodes, connections, parameter values, optional layout hints.
- `BuildTarget` (required):
  - `kind`: `toe` | `tox`
  - `output_path`: absolute path to resulting `.toe`/`.tox`
  - Optional `template`: a known-good template `.toe` or `.toe.dir` to start from.
- `ToolPaths` (optional):
  - `td_bin`: default `C:\Program Files\Derivative\TouchDesigner\bin`
  - `toeexpand`, `toecollapse`

## Outputs
- `ExpandedProjectPlan` (required):
  - `expanded_dir`: path to created/updated `*.toe.dir` or `*.tox.dir`
  - `toc_path`: path to sibling `.toc` file used by `toecollapse`
  - `writes`: list of file write operations (path + bytes/text description)
  - `deletes`: list of deletions (only if necessary; prefer non-destructive edits)
- `CollapseCommand` (required):
  - command line invoking `toecollapse.exe <base_name>`
  - **Important**: `toecollapse.exe` takes a *base filename*, not a directory. It expects:
    - `<base_name>.dir` (or `<base_name>.toe.dir`/`<base_name>.tox.dir` depending on how you name it)
    - `<base_name>.toc`
  - If you want the output to have `.toe`/`.tox` extension, include it in `<base_name>` (e.g. `my_project.toe`).
- `ValidationPlan` (required):
  - `toeexpand` the output
  - verify expected operators/connections exist (structural checks)

## Hard rules
- Must end in a loadable `.toe`/`.tox` generated by `toecollapse.exe`.
- Must preserve stable schemas: do not modify lossless/semantic JSON formats.
- Prefer "modify template" over "generate from scratch" to minimize TD file-format risk.
- Output must be deterministic and replayable: same inputs → same file plan.
- Do not invent file types or extra sidecar files (e.g. a `.network` file) unless you have verified TouchDesigner actually uses them.

## Recommended approach
1) Choose a template project compatible with the target TD build.
2) Expand template with `toeexpand.exe` (or start from an existing expanded dir).
3) Apply edits through the lossless JSON intermediate (recommended and safest):
   - Convert expanded dir → lossless JSON via `toe_to_json_LOSSLESS.py`
   - Apply `TDNetworkSpec` changes in JSON
   - Rebuild expanded dir via `json_to_dir_LOSSLESS.py`
4) Ensure `.toc` is correct and includes all files **in the exact format TouchDesigner expects**:
   - Preserve `.toc` header lines (many `.toc` files start with a `# ...` header).
   - Preserve ordering.
   - Use UTF-8 **without BOM** and Unix line endings (`LF` only).
5) Collapse via `toecollapse.exe <base_name>` (see above).
6) Expand output again for validation.
